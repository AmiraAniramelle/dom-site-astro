# Общие правила для КАЖДОЙ секции главной страницы

> Прочитай этот файл перед выполнением любой задачи по секциям.
> Также прочитай DOM-Architecture-Spec.md в корне репозитория.

## Правило 1: Визуально идентично
Скопируй ВСЕ Tailwind-классы и HTML-структуру из source-DOM.html.
Не меняй классы, не упрощай, не «улучшай».

## Правило 2: CMS-Ready
Все тексты, картинки, ссылки — через props.
Компонент НЕ содержит захардкоженный контент.

Пример — как должно быть:

БЫЛО (хардкод):
```html
<h2 class="font-messiri home-title text-dom-graphite">Миссия проекта DOM</h2>
```

СТАЛО (компонент):
```astro
---
interface Props { title: string }
const { title } = Astro.props
---
<h2 class="font-messiri home-title text-dom-graphite">{title}</h2>
```

Классы — те же. Текст — через переменную.

## Правило 3: Данные
Тексты из оригинала положи в src/data/pages/home.ts.
Экспортируй объект homePageData с ключом для каждой секции.
Если файл уже существует — ДОБАВЬ новую секцию, не удаляй существующие.

## Правило 4: SEO
- <h1> — ТОЛЬКО в Hero (секция 1). Больше нигде на странице.
- Все остальные секции используют <h2> для заголовков, <h3> для подзаголовков.
- Не пропускай уровни (нет h2 → h4).

## Правило 5: Ссылки
- onclick="navigateTo('path')" заменяется на <a href="/ru/path/">
- onclick="navigateToProduct('formula-money')" заменяется на <a href="/ru/products/formula-money/">
- Внешние ссылки: rel="noopener noreferrer" target="_blank"

## Правило 6: НЕ ДЕЛАЙ
- Не меняй Tailwind-классы на другие
- Не убирай градиенты, тени, анимации
- Не упрощай вложенность HTML
- Не меняй шрифты и размеры
- Не добавляй свои стили
- Не трогай другие компоненты и страницы

## Правило 7: Проверка
После каждой секции: npm run dev → открой localhost:4321/ru/ → секция визуально идентична оригиналу.

## Правило 8: Картинки
- Используй <Image /> из 'astro:assets' вместо <img> для всех картинок
- Импортируй: import { Image } from 'astro:assets'
- Для картинок из src/assets/: import myImage from '../../assets/images/photo.png'
  затем <Image src={myImage} alt="описание" />
- Для картинок по URL (CMS, внешние): используй <img> с обязательными width и height
- КАЖДАЯ картинка должна иметь осмысленный alt (не пустой, не "image")
- Hero-картинки: loading="eager"
- Все остальные: loading="lazy"

## Правило 9: Доступность (Accessibility)
- <nav> — обязательно aria-label="Главная навигация"
- Ссылки с картинкой без текста — обязательно aria-label
- Кнопки выпадающих меню — aria-expanded + aria-haspopup="true"
- Все интерактивные элементы доступны с клавиатуры
- Модальные окна — role="dialog" + aria-modal="true"
- Кнопки карусели/слайдера — aria-label="Предыдущий"/"Следующий"
