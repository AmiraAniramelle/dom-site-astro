# Общие правила для страницы «О методе»

> Прочитай DOM-Architecture-Spec.md в корне репозитория.
> Файл-источник: source-DOM/source-DOM.html
> Страница «О методе»: строки 5697–6177

## Правила (те же что для главной)

1. ВИЗУАЛЬНО ИДЕНТИЧНО: Скопируй ВСЕ Tailwind-классы и HTML-структуру. Не меняй, не упрощай.

2. CMS-READY: Все тексты, картинки, ссылки — через props. Данные в src/data/pages/method.ts

3. SEO: Один H1 на всю страницу (в первой секции). Остальные — H2, H3, H4.

4. ССЫЛКИ: onclick="navigateTo('...')" → <a href="/ru/.../">

5. НЕ ДЕЛАЙ: не меняй классы, не убирай анимации, не добавляй zoom/scale, не используй text-overflow: ellipsis

6. КАРТИНКИ: обычные <img> с alt, width, height, loading="lazy"

7. ACCESSIBILITY: aria-label на навигациях, aria-expanded на аккордеонах

8. МОБИЛЬНАЯ ВЕРСИЯ: убедись что ничего не вылезает за экран. Не используй фиксированные ширины. Все контейнеры max-width: 100%.

9. ДАННЫЕ: создай src/data/pages/method.ts с экспортом methodPageData. Если файл уже существует — обнови его, не удаляя существующие данные.

10. СТРАНИЦА: src/pages/ru/method.astro — использует BaseLayout с SEO meta, импортирует компоненты и данные.
